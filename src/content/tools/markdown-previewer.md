### Frequently Asked Questions (FAQs)

1. **What is Markdown?** A lightweight markup language with plain-text-formatting syntax designed so that it can be converted to HTML and many other formats.
2. **Is it the same as HTML?** No, Markdown is a simpler "shorthand" that gets converted into HTML for web display.
3. **Does it support images?** Yes, using the `![alt text](url)` syntax. Our previewer renders them instantly.
4. **What is GFM?** GitHub Flavored Markdown, an extension of the standard that adds support for tables, task lists, and autolinks.
5. **Can I use HTML inside Markdown?** Yes, most engines allow raw HTML for more complex layout needs that Markdown doesn't cover.
6. **How do I make a link?** Use the `[text](url)` syntax. It's the most professional way to handle internal and external references.
7. **What are "Frontmatter" blocks?** Metadata (usually YAML) at the top of a file used by static site generators to define titles, dates, and layouts.
8. **How precise is our previewer?** We use the industry-standard "Marked" or "Remark" logic to ensure your output is 100% compliant with GFM.
9. **Is Markdown good for SEO?** Yes, it encourages a "Semantic Heading Structure" which is a critical signal for search engine bots.
10. **Is my data safe?** Yes, our tool works entirely offline in your browser; your sensitive proprietary docs never leave your computer.
