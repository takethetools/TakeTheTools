### Frequently Asked Questions (FAQs)

1. **What is Base32 Encoding?** A method of representing binary data as a string of text using a restricted set of 32 characters (A-Z and 2-7).
2. **How does it differ from Base64?** Base32 uses a smaller alphabet and is more human-readable because it excludes confusing characters (like 0, 1, 8, and 9) and is case-insensitive.
3. **Why do we use it for 2FA?** It was chosen for the Google Authenticator standard because users can easily type it on a keyboard without errors.
4. **Is it an encryption?** No. Encoding is just a way to change the format of data; it is not a secure "secret" and can be easily reversed.
5. **Does it increase file size?** Yes, Base32 encoding increases the size by about 60% (whereas Base64 only increases it by 33%).
6. **What is the "Padding"?** The `=` signs at the end of the string are used to fill out the last block of data to the required length.
