## 1. The Mathematical Foundation: Understanding the ISO 32000 Standard

To understand how PDFs merge, we must first look at the **PDF Specification (ISO 32000)** and the four components of a PDF file.

### 1.1 The Four-Stage Architecture
Every PDF is composed of:
1. **The Header**: Identifying the PDF version (e.g., `%PDF-1.7`).
2. **The Body**: Containing all the visible and invisible "Objects" (text, fonts, images).
3. **The Xref Table**: The "Compass" that tells the viewer where every object is located in the file.
4. **The Trailer**: Providing the roadmap to the Xref table and the document "Root."

### 1.2 The Mapping Challenge
When you merge multiple PDFs, you aren't just "Gluing" them together. The software must perform a "Global Re-indexing." If File A and File B both have an object labeled "1 0 obj," the merger must rewrite these into a unique sequence (e.g., Object 1, Object 2, etc.) and update every single reference in the newly generated **Cross-Reference Table**.
