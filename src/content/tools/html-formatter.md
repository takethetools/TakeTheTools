## 1. The Mathematical Foundation: Recursive Descent Parsing

To understand how a formatter works, we must first look at the **Mechanics of the Document Object Model (DOM)**.

### 1.1 The Hierarchical Tree Strategy
HTML is not a flat file; it is a "Tree."
- **The Transformation**: A formatter works by parsing the raw string into a "Memory Tree" using a recursive descent algorithm. 
- **The Logic**: As the tool moves "Down" the tree into a child element (like a `<li>` inside a `<ul>`), it increments the **Indentation Factor**. As it moves "Up," it decrements it.
Our engine performs this "Tree Traversal" with 100% precision, ensuring every tag is perfectly aligned at the correct depth—a hallmark of elite systems engineering.

### 1.2 Handing "Void" and "Self-Closing" Tags
In 20/26, the distinction between a `<div>` (requires closing) and an `<img>` (void) is critical. Our tool utilizes the latest W3C specifications to identify these tags automatically, preventing "Ghost Content" caused by incorrect tag wrapping.
