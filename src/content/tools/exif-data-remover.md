## 1. The Mathematical Foundation: Understanding the EXIF Structure

To understand how metadata is removed, we must first look at the **Binary Anatomy of the JPG/TIFF File**.

### 1.1 The APP1 Marker and JPEG Headers
A JPEG image is not just pixels; it is a series of "Segments."
- **The Protocol**: Metadata is stored in a segment called **APP1** (Application Segment 1). 
- **The Marker**: This segment starts with the hexadecimal signature `0xFFE1`.
- **The Task**: Our engine scans the binary stream for this marker, identifies the length of the metadata block, and physically "Excises" it from the file stream while preserving the image signal (the SOS segment).

### 1.2 The TIFF-Based Tag-Value Pairs
EXIF data follows the TIFF (Tagged Image File Format) specification. 
- **The Data Type**: Each piece of info (like "Shutter Speed") is a "Tag" with a specific numerical ID (e.g., `0x829A`). 
- **The Scrubbing Logic**: Our tool doesn't just "Blank" the values; it removes the entire **IFD (Image File Directory)**, ensuring that no "Ghost Blocks" remain that could be reconstructed by forensic software.
