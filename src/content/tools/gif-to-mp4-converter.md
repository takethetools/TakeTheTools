## 1. The Mathematical Foundation: LZW vs. DCT

To understand why MP4 is superior, we must first look at the **Mathematics of Compression**.

### 1.1 The GIF Legacy: LZW (Lempel-Ziv-Welch)
GIF uses a format called LZW, which is a "Dictionary" approach to lossless data.
- **The Limit**: GIF saves every single pixel of every single frame (up to 256 colors). 
- **The Result**: For a 5-second animation at 30fps, a GIF stores 150 individual images. This is "Linear Growth" data which is inefficient for modern web distribution.

### 1.2 The MP4 Revolution: Inter-Frame Prediction
MP4 (specifically using H.264/H.265) doesn't save every frame.
- **I-Frames (Keyframes)**: A full image.
- **P-Frames (Predicted)**: Only the *changes* from the previous frame.
- **B-Frames (Bi-directional)**: Changes from both the past and the future.
- **The Math**: By only storing the "Delta" (the movement), MP4 achieves a 10x to 50x reduction in file size without visible quality loss. Our engine performs this "Inter-Frame Synthesis" with 100% precision.
